# Phase 0 — Methodology Lock

## Goal

Lock the analytical foundation before application development starts.

The engine must be deterministic, explainable, and evidence-linked.

## Locked correction

The original rough rule:

```text
A/C = exploration
B/D = safety-control
```

is rejected as an engine rule.

Reason: A/B/C/D meanings change by question.

Examples:

- Q2A = direct curiosity / risk
- Q13A = abstract meaning / philosophy
- Q15A = structure / control
- Q18A = competitive reward

Locked model:

```text
Answer → option-specific tags → weighted scores → axis scores → contradictions → archetype → report seed
```

## Six core axes

1. Exploration vs Safety-Control
2. Thinking Style
3. Relationship and Distance Pattern
4. Agency, Leadership, and Control
5. Ambiguity and Fear Response
6. Deep Motive Structure
