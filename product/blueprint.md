# Product Blueprint

## Core thesis

The 20 Corridors is a symbolic decision-pattern game where users answer 20 scenario-based A/B/C/D questions and receive a detailed reflective analysis.

The product value is not the question list alone. The product value is the analysis engine:

```text
Repeated choices + contradictions + motive signals = specific reflective report
```

## Non-negotiable product statement

> The 20 Corridors analyzes symbolic choices through deterministic, answer-specific tags. Its value comes from repeated patterns, contradictions, and motive signals. The engine must stay explainable, the report must stay evidence-linked, and the product must never present itself as a clinical psychological assessment.

## Primary output

Each completed game must produce:

- archetype
- dominant tags
- six-axis analysis
- contradiction map
- deep motive
- strengths
- failure modes
- falsifier / disproven-if section
- shareable summary seed

## Strategic risk

The product fails if results feel generic.

Bad report language:

> You are mysterious and deep.

Good report language:

> Your choices show curiosity filtered through observation. This appears when you approach unknown objects indirectly but still choose meaning-rich or exploratory paths.

## Development order

1. Lock methodology
2. Build deterministic scoring engine
3. Test golden profiles
4. Build basic quiz UI
5. Build result report
6. Add cinematic design
7. Add share card
8. Add feedback and analytics
9. Add backend persistence
10. Add friend comparison
11. Add AI report expansion
12. Add monetization

## MVP boundary

The MVP must include:

- 20-question flow
- deterministic scoring
- result overview
- detailed report
- shareable result summary
- feedback signal

The MVP must not start with:

- accounts
- payments
- AI-generated interpretation
- friend comparison
- database dependency
- complex admin panel
