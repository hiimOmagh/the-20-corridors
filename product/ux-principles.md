# UX Principles

## Visual direction

Use a dark cinematic corridor aesthetic with modern glass-like cards.

Desired feeling:

- mysterious
- premium
- reflective
- fast
- mobile-first
- serious but not clinical

Avoid:

- childish quiz UI
- BuzzFeed-style visuals
- medical/diagnostic visuals
- fake occult overdesign
- overwhelming animation

## Game flow

1. Landing page
2. Method / disclaimer preview
3. Instructions
4. 20-question corridor flow
5. Calculation transition
6. Result overview
7. Full report
8. Share card
9. Feedback

## Question UX rules

- One question per screen.
- Always show progress: `Corridor 07 / 20`.
- Force one answer: A/B/C/D.
- No "depends" option.
- Large mobile tap targets.
- Keyboard shortcuts on desktop: A/B/C/D.
- Optional short undo window.
- No login before result.
- Do not show scoring hints during the quiz.

## Result UX rules

The result must not be one long text wall.

Use:

- archetype hero card
- dominant-trait chips
- axis cards
- contradiction cards
- strengths / failure modes
- falsifier box
- share card preview

## Calculation screen

Use a short transition after the final answer:

- Mapping corridors
- Detecting dominant signals
- Checking contradictions
- Building report

This creates perceived depth without pretending the system is clinical.
